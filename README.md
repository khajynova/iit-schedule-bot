# 🎓 Бот расписания ИИТ БГУИР

Telegram-бот для отслеживания расписания преподавателей ИИТ БГУИР.

## 📌 Возможности

- 📅 Просмотр расписания на сегодня, завтра и неделю
- 🔔 Автоматические уведомления об изменениях в расписании
- 👨‍🏫 Поддержка нескольких преподавателей
- 📊 Статистика по занятиям
- ⚙️ Настройка уведомлений
- 🎯 Фильтрация занятий только для выбранного преподавателя

## 🚀 Быстрый старт

### Для пользователей

1. Найдите бота в Telegram: `@ваш_username_бота`
2. Отправьте команду `/start`
3. Установите преподавателя: `/set_teacher Иванов А.А.`
4. Готово! Используйте команды для просмотра расписания
   /start Начать работу с ботом
   /set_teacher ФИО Установить преподавателя (например, /set_teacher Иванов А.А.)
   /today Расписание на сегодня
   /tomorrow Расписание на завтра
   /week Расписание на неделю
   /settings Настройки бота
   /stats Статистика по занятиям
   /remove_teacher Удалить преподавателя
   /help Помощь

### Для разработчиков (локальный запуск)

#### Требования

- Python 3.10+
- Git
- Telegram Bot Token (получить у [@BotFather](https://t.me/BotFather))

#### Установка

```bash
# 1. Клонируйте репозиторий
git clone https://github.com/khajynova/iit-schedule-bot.git
cd iit-schedule-bot

# 2. Создайте виртуальное окружение
python -m venv venv

# 3. Активируйте виртуальное окружение
# Windows:
venv\Scripts\activate
# Linux/Mac:
source venv/bin/activate

# 4. Установите зависимости
pip install -r requirements.txt

# 5. Создайте файл .env с токеном
echo TELEGRAM_BOT_TOKEN=ваш_токен_здесь > .env

# 6. Запустите бота
python bot.py
```

Деплой на Render.com

Загрузите код на GitHub

Зарегистрируйтесь на Render.com

Подключите GitHub репозиторий

Добавьте переменную окружения: TELEGRAM_BOT_TOKEN

Нажмите "Create Worker"

Render автоматически:

Установит зависимости из requirements.txt

Запустит бота с помощью python bot.py

Будет перезапускать бота при каждом git push

Переменные окружения
Для работы бота необходима одна переменная:

Переменная Описание
TELEGRAM_BOT_TOKEN Токен бота от @BotFather
Настройка UptimeRobot (чтобы бот не засыпал)
На бесплатном тарифе Render бот "засыпает" через 15 минут бездействия. Чтобы этого избежать:

Зарегистрируйтесь на UptimeRobot

Добавьте монитор:

Monitor Type: HTTP(s)

URL: https://iit-schedule-bot.onrender.com/health

Interval: 5 minutes

🐛 Решение проблем
Бот не запускается
bash

# Проверьте установку зависимостей

pip install -r requirements.txt

# Проверьте наличие токена в .env файле

cat .env

# Проверьте логи в Render (вкладка "Logs")

Конфликт при запуске (Conflict)
Ошибка Conflict: terminated by other getUpdates request означает, что бот уже запущен где-то еще. Решение:

Остановите локального бота (закройте терминал)

На Render добавлен автоматический сброс вебхука при запуске

Бот не отвечает
Проверьте, запущен ли бот на Render (вкладка "Logs")

Проверьте, не истек ли токен (получите новый у @BotFather)

Проверьте UptimeRobot (если бот заснул)

База данных не создается
bash

# Создайте папку вручную

mkdir data
chmod 755 data # На Linux/Mac

📊 База данных
Бот использует SQLite для хранения данных:

Таблицы
users - пользователи и их преподаватели

schedule_cache - кеш расписания

schedule_changes - история изменений

Резервное копирование
bash

# Скопируйте файл базы данных

cp data/schedule*bot.db data/backup*$(date +%Y%m%d).db
🔒 Безопасность
Токен бота хранится в .env файле (не в коде!)

.env добавлен в .gitignore

База данных не содержит паролей

Используется только публичное API БГУИР
